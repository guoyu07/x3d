A.1.2 Introduction

An XML Document Type Definition (DTD) defines syntax of an XML tagset [XML].

The X3D DTD has sufficient expressive power to:

  • Express scenes that may include any or all aspects of the functionality specified in ISO/IEC 19775-1.
  • Validate correct naming and usage of elements (i.e., X3D nodes)
  • Identify correct naming of corresponding attributes (i.e., X3D fields)
  • Selectively validate content according to the defined profiles

The X3D DTD does not have sufficient expressive power to:

  • Validate correct types or ranges of attribute values (i.e., X3D fields)
